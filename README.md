 # Projektarbeit Modul 183
Noé, Samuel, Noël

Name der WebApp: NSN8
Kurzbeschreibung:
Unsere Projektidee im Modul 183 ist eine WebApp für einen Kleidershop namens NSN8. Kunden sollen Kleidung ansehen und auswählen, und eventuell Rezensionen hinzufügen und sich Produkte merken können.Zusätzlich sollen Benutzerkonten und evtl. Rollen umgesetzt werden, damit wir die Sicherheit der WebApp praktisch zeigen können.

Features mit Prio:
1. Login, Rollen, Produkte hinzufügen/ bearbeiten, Accounts
2. Rezensionen, Checkout, Warenkorb, Merkliste,

Begründung der Features:
Die Features mit Priorität 1 wurden gewählt, weil sie die Grundlage der WebApp bilden. Login und Accounts sind nötig, damit Benutzer eindeutig erkannt werden können. Rollen sind wichtig, damit normale Kunden und Administratoren unterschiedliche Rechte haben. Die Produktverwaltung braucht es, weil ein Kleidershop ohne Produkte nichts bringt. Zudem ist es gut geeignet für verschiedene OWASP Themen.

Die Features mit Priorität 2 wurden gewählt, weil sie typische Funktionen eines Online-Shops sind. Diese Funktionen sind aber weniger wichtig als Login, Rollen und Produktverwaltung, darum haben sie eine tiefere Priorität.

Unsere OWASP:
1. Software Supply Chain Failures
2. Injection
3. Broken Access Control
4. Cryptographic Failures

Techstack:
Frontend	HTML, CSS, JavaScript 
Backend	Java / Spring
Datenbank	MYSQL

Begründung des Techstacks:
HTML, CSS und JavaScript wurden für das Frontend gewählt, weil damit eine einfache und übersichtliche Benutzeroberfläche für den Kleidershop erstellt werden kann. Java / Spring wurde für das Backend gewählt, weil sich damit Login, Rollen, Validierung und Datenbankzugriffe strukturiert umsetzen lassen.

MYSQL haben wir gewählt, weil Produkte, Konten, Warenkorb und Rezensionen als strukturierte Daten gespeichert werden.
